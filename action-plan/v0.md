> **Version:** 0.1.0

## What Was Built in This Iteration

This iteration delivers the **plugin skeleton and infrastructure** — everything needed so that the next developer (or AI) can pick up any module and start implementing it immediately.

### 1. Plugin Bootstrap (`store-kit.php`)

- Standard WordPress plugin header with WooCommerce headers (`WC requires at least`, `WC tested up to`)
- Defines constants: `JWP_STK_VERSION`, `JWP_STK_PLUGIN_FILE`, `JWP_STK_PLUGIN_URL`, `JWP_STK_PLUGIN_DIR`, `JWP_STK_PLUGIN_BASE`
- **WooCommerce dependency check** — shows admin notice if WooCommerce is not active
- Bootstraps on `plugins_loaded` via `JWP_STK\Plugin::get_instance()`
- Declares **HPOS (High-Performance Order Storage)** compatibility

### 2. Options Engine (`includes/class-options.php`)

Follows the exact same pattern as RSFV's `class-options.php`:

- Singleton class `JWP_STK\Options`
- Single serialized WP option: `jwp_stk_options`
- Methods: `get($key, $default)`, `set($key, $value)`, `has($option)`, `delete($key)`
- All plugin settings (modules, theme engine, future module settings) stored here

### 3. Module Manager (`includes/class-module-manager.php`)

- Singleton class `JWP_STK\Module_Manager`
- Registers all **17 modules** across 3 tiers:
  - **Core** (6): Quick View, Cart Notices, Checkout Field Editor, Custom Order Statuses, Stock Manager, Product Tab Manager
  - **Growth** (6): Wishlist, Free Shipping Bar, Variation Swatches, Sale Badges & Labels, Product Bundles, Ajax Product Search
  - **Power** (5): Points & Rewards, Bulk Pricing & Tiered Discounts, Back-in-Stock Waitlist, Advanced Reviews, Order & Customer Export
- Each module has: `title`, `description`, `tier`, `default` (all default to `false`)
- Active modules stored in `jwp_stk_options['active_modules']` as array of slugs
- Methods: `get_modules()`, `get_modules_by_tier()`, `is_module_active($slug)`, `get_active_modules()`, `activate_module($slug)`, `deactivate_module($slug)`
- Filterable via `jwp_stk_registered_modules`

### 4. Admin Dashboard — React App

#### PHP Side

- **`includes/Admin/class-register.php`** — Registers under the shared "JetixWP" admin menu (same pattern as RSFV), submenu slug `jwp-stk-dashboard`
- **`includes/Admin/class-admin-dashboard.php`** — Enqueues the React bundle from `assets/js/dashboard/`, localizes `jwpStkDashboard` with nonce, apiBase, pluginUrl, version
- **`includes/Admin/Views/html-admin-dashboard.php`** — Mount point `<div id="jwp-stk-dashboard-app">`

#### React Side (`src/dashboard/`)

- **`index.js`** — Entry point, creates React root
- **`App.js`** — Tab-based layout with two tabs: **Modules** and **Settings**
- **`components/ModulesPanel.js`** — Fetches modules via REST API, displays them grouped by tier (Core/Growth/Power) in a responsive card grid, each with a toggle switch for activation/deactivation
- **`components/SettingsPanel.js`** — Theme Compatibility Engine selector (Disabled / Auto / Default / Storefront) with save button
- **`style.css`** — Clean, WordPress-admin-native styling with tier badges (blue/green/purple), card grid, active state borders

#### Build

- Uses `@wordpress/scripts` (v30.13) — no custom webpack config
- Source: `src/dashboard/` → Output: `assets/js/dashboard/`
- Commands: `npm run build`, `npm run start` (for dev watch mode)
- Dependencies: `@wordpress/api-fetch`, `@wordpress/components`, `@wordpress/element`, `@wordpress/i18n`

### 5. REST API (`includes/Admin/class-rest-api.php`)

Namespace: `jwp-stk/v1`

| Method | Endpoint            | Purpose                                        |
|--------|---------------------|-------------------------------------------------|
| GET    | `/modules`          | List all modules with active state              |
| POST   | `/modules/toggle`   | Toggle a module on/off                          |
| GET    | `/settings`         | Get settings + available theme engines          |
| POST   | `/settings`         | Save settings (allowlisted keys only)           |

- All routes require `manage_options` capability
- Settings saving uses an explicit allowlist (`jwp_stk_allowed_settings_keys` filter)
- Extensible via `jwp_stk_register_rest_routes` action

### 6. Theme Compatibility Engine

Follows the RSFV pattern exactly:

- **`includes/Compatibility/class-theme-provider.php`** — Engine hub
  - `get_theme_engines()` — Returns filterable array of engines (`jwp_stk_theme_compatibility_engines`)
  - `load_theme_compat()` — Runs on `after_setup_theme`, detects theme, loads appropriate engine
  - Supports: `disabled`, `auto` (detects current theme), or manually selected engine
  - Strips `-child` suffix for child theme detection
  - Falls back to `default` if theme not in registry
- **`includes/Compatibility/Themes/class-base-compatibility.php`** — Abstract base class
  - Late static binding singleton
  - Adds `jwp-stk-active` and `jwp-stk-theme-{id}` body classes
  - Provides `get_current_dir_url()` and `get_current_dir()` helpers
- **Engines built:**
  - `default` (Fallback) — `includes/Compatibility/Themes/Fallback/`
  - `storefront` — `includes/Compatibility/Themes/Core/Storefront/`
- Each engine has its own `class-compatibility.php` and `styles.css`

### 7. Conventions & Prefixes

| Context           | Prefix         |
|-------------------|----------------|
| PHP namespace      | `JWP_STK\`    |

| Asset handles      | `jwp-stk-`    |

## What's Remaining for Free Version MVP

### Module Implementations (the actual features)

None of the 17 modules have their feature code built yet. Each needs:

1. **Quick View** [Core] — Lightbox modal, AJAX product load, variable product support, trigger button customization
2. **Cart Notices** [Core] — Free shipping progress bar, rule-based notices, countdown timers, placement options
3. **Checkout Field Editor** [Core] — Drag-and-drop ordering, custom field types, conditional logic, validation rules
4. **Custom Order Statuses** [Core] — Unlimited statuses, colour-coded columns, per-status emails, bulk actions
5. **Stock Manager** [Core] — Inline editable stock table, bulk update, per-product thresholds, CSV export
6. **Product Tab Manager** [Core] — Custom global/per-product tabs, drag-and-drop, WP editor content, hide defaults
7. **Wishlist** [Growth] — Guest + logged-in support, shareable URL, add-to-cart from wishlist, header widget count
8. **Free Shipping Bar** [Growth] — Sticky/inline display, threshold messaging, auto-update on cart change, multi-zone
9. **Variation Swatches** [Growth] — Colour/image/button swatches, global defaults, out-of-stock styling, tooltips
10. **Sale Badges & Labels** [Growth] — Auto % discount, custom label rules, shape/colour/position, scheduling
11. **Product Bundles** [Growth] — Combine products, optional discount, stock sync, bundle display
12. **Ajax Product Search** [Growth] — Live search, thumbnail/price/category results, configurable count, SKU search
13. **Points & Rewards** [Power] — Points per order, points for reviews/sign-ups, cart discount redemption, My Account
14. **Bulk Pricing & Tiered Discounts** [Power] — Quantity tiers, per-product/category rules, pricing table, coupon stacking
15. **Back-in-Stock Waitlist** [Power] — One-click sign-up, auto email on restock, admin waitlist count, guest support
16. **Advanced Reviews** [Power] — Photo/video uploads, verified badge, helpful votes, star rating filter
17. **Order & Customer Export** [Power] — Custom columns, date/status/product filters, scheduled exports, CSV/XLSX

### Per-Module Architecture Pattern

Each module should follow this pattern:
```
includes/Modules/{ModuleSlug}/
├── class-module.php          # Module bootstrap, hooks into WC
├── class-frontend.php        # Frontend output (if applicable)
├── class-admin.php           # Admin UI (if applicable)
└── assets/                   # Module-specific CSS/JS
```

The module loader in `class-plugin.php` should be extended to conditionally load active modules after the Module_Manager initializes.

### Infrastructure Still Needed

- [ ] **Module autoloader** — Dynamic loading of active modules from `includes/Modules/`
- [ ] **Per-module settings** — Each module will need its own settings section in the React dashboard. The REST API `settings` endpoints support this via the options engine, but the React UI needs module-specific settings panels
- [ ] **Frontend asset loading** — Conditional enqueue of module-specific CSS/JS only when that module is active
- [ ] **Uninstall hook** — `uninstall.php` to clean up `jwp_stk_options` and any other data on plugin deletion
- [ ] **Activation/Deactivation hooks** — `register_activation_hook` / `register_deactivation_hook` for setup/cleanup
- [ ] **More theme engines** — Only Default and Storefront are built. Add support for: Astra, Kadence, Divi, GeneratePress, Neve, Flatsome, OceanWP, etc. (follow the RSFV pattern — each theme gets a folder with `class-compatibility.php` + `styles.css`)
- [ ] **Plugin compatibility engine** — Like RSFV's `Plugin_Provider`, auto-detect and load compatibility for popular plugins (Elementor, Divi Builder, etc.)
- [ ] **Translation files** — `.pot` file generation for the `jwp-stk` text domain
- [ ] **Admin dashboard enhancements** — Module-specific settings panels, status indicators, onboarding wizard
- [ ] **WP.org readme.txt** — Standard WordPress.org readme with screenshots, FAQ, changelog
- [ ] **WPCS compliance audit** — Run PHP_CodeSniffer with WordPress-Coding-Standards ruleset across all PHP files

### For WP.org Free Version

The free version should ship with all **Core** modules (6) fully functional, plus a selection of **Growth** modules. **Power** modules can be listed but marked as "Pro" to drive upgrades.

Recommended free version scope:
- All 6 Core modules fully built
- 2-3 Growth modules (Wishlist, Free Shipping Bar, Variation Swatches recommended as highest-demand)
- Power modules visible but locked with upgrade CTA

## Key Filters & Actions

| Hook | Type | Description |
|------|------|-------------|
| `jwp_stk_loaded` | Action | Fires when plugin is fully loaded |
| `jwp_stk_registered_modules` | Filter | Modify registered modules array |
| `jwp_stk_theme_compatibility_engines` | Filter | Add/modify theme engines |
| `jwp_stk_register_rest_routes` | Action | Register additional REST routes |
| `jwp_stk_dashboard_localized_data` | Filter | Modify data passed to React app |
| `jwp_stk_rest_settings` | Filter | Modify settings returned via REST |
| `jwp_stk_allowed_settings_keys` | Filter | Allowlist additional saveable keys |
| `jwp_stk_body_classes` | Filter | Modify body classes added by theme engine |
